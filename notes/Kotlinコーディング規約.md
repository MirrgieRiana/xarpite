# Xarpite Kotlin コード（`src/commonMain/kotlin`）コーディング規約

これまでの実験であぶり出された規約を、1行1規約のリストにまとめたものです。
`escapeJsonString`・`MatrixPositionCalculator`・識別子名予想・`Formatter.format` などの各実験で判明した規約を統合・再編しています。

## 型注釈と式

1. 関数の戻り値型注釈は省略し、式本体で型推論に委ねる（`fun … = …`）。プロパティの型注釈も同様に省略する。
2. 中間変数を最小化し、標準ライブラリ（`binarySearch` 等）と `.let { }` で途中結果に名前を付けずに式1本へ畳み込み、`return` に直接埋め込む。スコープ関数のなかでは `let` が主力。
3. 複数文を要するプロパティ初期化は `buildList { … }` でなく `run { val list = …; list }` で書く（`buildList` は0件）。
4. 複数の値の組を生む大きな分岐は、宣言を先に出して各枝で代入するのでなく、`if`/`when` 式そのものの結果を `val (a, b) = …` の分解宣言で受ける。
5. 分岐の各枝はチェーン式を丸ごと書き切り、共通部分を手前にくくり出さない。

## 制御構造とイディオム

6. `for` 文は使わず `forEach`/`forEachIndexed` に統一する。`continue` 相当も `return@forEach` で表す。`for` は Channel 消費やネイティブのグルーコードなど、forEach で書けない箇所のみに残す。
7. 早期 return には elvis 演算子を好む（`getOrNull(…) ?: return …`）。
8. 途中の文字列変換は `String` の拡張関数として定義し、`.a().b().c()` とメソッドチェーンで一列に流す（引数渡しの入れ子呼び出しにしない）。

## レイアウトと改行

9. メソッドチェーンはレシーバ（`this` 等）を単独行に置き、後続の `.method` をすべて同一インデントの行頭ドットで揃える。
10. 行の長さでは折り返さない（リンタ設定は無く、1つの文法規則などは何文字でも1行に収める）。代わりに、メソッド呼び出しの区切りなど意味の境界では積極的に折る。
11. クラス本体は `{` の直後に空行を置かずに始める。
12. `require(…)` のメッセージラムダは長くても1行に保ち、`require(…)` の後は1行空けて主処理に入る。

## 命名

13. ローカル変数・プロパティには型名を埋め込まず、囲うクラスや型が与える文脈に委ねた短名にする（`receiver`/`left`/`right`）。ローカル変数は型名の lowerCamelCase をそのまま使う。
14. AST ノードのクラスは、演算の意味語でなく構文・トークンの見た目で命名する（`Decrement` でなく `MinusMinus`、丸括弧は `Round`、角括弧は `Square`）。
15. 演算クラスは役割を表すサフィックス（`Getter`/`Setter`/`Comparator`）を固定し、接頭は具体語にする（`Until` でなく `Exclusive`、判定系は `Comparator` 家系に分類する）。
16. 値の変換は逆変換でも `parse`/`from` を使わず、`to○○As○○` の形で方向を統一する。
17. 動詞選択が独特である（探索は `get`、呼び出しは `call`、整列は実装名 `mergeSort`、中核 API も超短い `get`/`run`、真偽を返す関数でも `is` を付けない）。
18. Map を保持するプロパティは `○○Table`、生成用のラムダは `○○Provider` とサフィックスを固定する。
19. 識別子は略さないフルワードを主流とし、確立した略語（`sb`/`ch`/`env`/`inc`）だけを許容する（`context` は544件すべてフルスペルで `ctx` は0件）。
20. `a to b` は「キー→値」の写像エントリ、`Pair(a, b)` は対等な2値のタプルとして意味論的に使い分ける。

## 構造とプラットフォーム

21. `commonMain` では `java.*` を import せず、多倍長小数は ionspin の BigDecimal、絶対値は単項マイナスなど、マルチプラットフォームで使える手段に揃える。
22. 汎用的なエラーは専用の例外クラスを新設せず、既存の `IllegalArgumentException`/`FluoriteException`/`RuntimeException` で投げ分ける。
23. 単なる値の入れ物のために `data class` を増やさない（全193クラス中4件）。

## 弱い規約・傾向

24.〔弱〕`if`/`when` の分岐の各枝に、その条件を説明する日本語の行コメント（`//`）を添えることがある（`commonMain` の `//` 付き日本語コメントは104件）。
